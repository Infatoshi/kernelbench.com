I need a full causal FlashAttention forward kernel in CUDA for the H100 SXM (SM90 Hopper, HBM3, bf16 tensor cores). The reference implementation is in reference.py and your kernel goes in solution.py. You can make whatever mess you want in this directory — scratch files, intermediate kernels, build artifacts, profiling traces — but the final answer has to be in solution.py with the same Model, get_inputs, and get_init_inputs interface as reference.py.

The op is standard multi-head causal attention: q, k, v of shape (B, H, S, D) bf16, scores = q @ k^T / sqrt(D) with a causal mask, softmax in fp32, times v, output (B, H, S, D) bf16. Correctness tolerance is 0.05 abs/rel on bf16 against the fp32-softmax reference. The shapes you have to handle are B=1 H=32 S=4096 D=128 (7B-class prefill), B=1 H=32 S=4097 D=128 (odd S, so your tiles get a ragged tail), B=4 H=8 S=8192 D=128 (batched long prefill), B=8 H=16 S=2048 D=64 (D=64 tile geometry), and B=1 H=8 S=16384 D=128 — at that last one an O(S^2)-memory approach cannot run at all, so you need the real thing: tiled streaming over KV with online softmax, never materializing the score matrix.
