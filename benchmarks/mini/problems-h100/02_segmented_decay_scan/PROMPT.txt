I need a fast segmented decay scan for the H100 SXM (SM90 Hopper, HBM3, 3.35 TB/s). The reference implementation is in reference.py and your kernel goes in solution.py. You can make whatever mess you want in this directory — scratch files, intermediate kernels, build artifacts, profiling traces — but the final answer has to be in solution.py with the same Model, get_inputs, and get_init_inputs interface as reference.py.

The op is a gated linear recurrence with per-token episode resets: h_t = x_t where resets[t] is true, otherwise h_t = decay_t * h_{t-1} + x_t, with h_{-1} = 0, and the output is the full trajectory h. Inputs are x of shape (B, T, D) bf16, decay of shape (B, T, D) fp32 with values in (0, 1), and resets of shape (B, T) bool; output is (B, T, D) bf16. Correctness tolerance is 0.05 abs/rel on bf16 — long decay chains accumulate rounding against the fp32 reference loop. The shapes you have to handle are B=8 T=4096 D=2048 (batched trainer shape), B=4 T=8191 D=1024 (odd T, so chunked strategies get a ragged tail), B=16 T=2048 D=4096 (wide channels), and B=1 T=16384 D=512 (a single long sequence, so batch parallelism alone won't save you — you have to parallelize over T).

This needs to be a real custom kernel — the sequential reference loop is thousands of launches of nothing. The recurrence is associative once you fold the reset into the decay, so a parallel or chunked scan works, but the textbook tl.associative_scan and cumprod-of-decays recipes don't handle the reset mask as written (and a global cumprod underflows at these lengths anyway) — you have to derive the combine operator yourself.
