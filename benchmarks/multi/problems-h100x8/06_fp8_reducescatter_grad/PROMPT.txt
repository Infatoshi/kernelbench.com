I need you to write a fast fp8 gradient reduce-scatter for an 8xH100 SXM node (NVLink4, ~900 GB/s/GPU, NVSwitch). Runs across all 8 GPUs under torchrun. Reference is in reference.py, your kernel goes in solution.py — same Model, get_inputs, get_init_inputs interface, must work at world_size=8. Mess is fine; final answer in solution.py.

The op: FSDP/ZeRO gradient reduction with fp8 compression. Each rank produces a full (tokens, hidden) bf16 gradient. You must quantize each rank's gradient to fp8_e4m3, sum the fp8-compressed gradients across all 8 ranks, and scatter the result so each rank keeps tokens/world rows of the summed gradient (a reduce-scatter), returned as bf16. The wire payload is fp8 — 1 byte/elem, half a bf16 reduce-scatter. Read reference.py: it models the per-rank fp8 round-trip so the tolerance already accounts for compression error.

Graded purely on busbw — fraction of NVLink peak, geomean over shapes — and the byte budget is the fp8 (1-byte) volume, so you only get credit if the wire really carries fp8. Forbidden: every torch.distributed collective (reduce_scatter, all_reduce, all_gather, all_to_all, the c10d/functional variants). Implement the reduce-scatter yourself over NVLink: a reduce-scatter ring/halving in fp8, CUDA symmetric memory (peers accumulate each other's fp8 shards), or NVSHMEM. The subtlety is reducing in fp8 without blowing past tolerance — pick where you up-cast to accumulate. Look up symmetric-memory / NVSHMEM docs yourself.

Correctness tolerance on the bf16 output is 5e-2 abs/rel (fp8 compression), across every shape, 5 rank-distinct trials, and numeric-stress rescales. Shapes (tokens must divide across the 8 ranks): tokens=4096 hidden=8192 (large), tokens=512 hidden=8192 (transitional), tokens=64 hidden=8192 (small, latency-bound), tokens=4096 hidden=8191 (off-alignment hidden).
