# KernelBench-Mega Leaderboard

The deck is two megakernel problems. The published, GPU-scored board is
`02_kimi_linear_decode` (Kimi-Linear W4A16 decode megakernel) across three
GPUs. `01_rl_grid_ppo` has reference/check/prompt in place but no published
board yet.

Source of truth is `public/data/mega/results.csv` (rendered live at `/mega`),
regenerated by `scripts/build_mega_leaderboard.py` from the run archives. The
builder requires a per-run `gpu` marker and auto-excludes any run whose agent
transcript references another run's archive (cross-run contamination tripwire).

## Source data

- `public/data/mega/results.csv` — machine-readable board the `/mega` page reads.
- `outputs/runs/<run_id>/` — per-run artifacts (transcripts, solution.py, result.json). Local only; gitignored.
- `DEVLOG.md` — running record of design decisions, dead ends, and lessons.
