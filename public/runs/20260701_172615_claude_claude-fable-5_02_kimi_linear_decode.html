What solution.py is

One cooperative CUDA __global__ (via load_inline), launched exactly once per step() — 188 blocks × 512 threads, 14 grid-wide sense-reversing barriers per token, with finer dependencies handled by producer-consumer spin counters inside stages (KDA state update spins on per-head q/k/v/g readiness; the MoE router gates routed-expert GEMVs; expert down-projections spin on per-expert silu(gate)·up completion; MLA q_abs and output-absorb spin similarly). Everything is fused: all int4 dequant-GEMVs, conv+silu, gated-delta state update, absorbed-latent MLA attention with two streaming softmax passes, MoE top-8 + 9 expert FFNs, RMSNorms, residuals, and the one-time cache ingest (so the growing MLA cache never needs a torch.cat — the Model owns a capacity buffer and returns views).

Three findings did most of the work:

  1. Exact-rounded SIMD dequant. The reference rounds each dequantized weight to bf16 before its fp32 GEMV. Skipping that (a fused fp32 group-fold) leaves ~0.4% output noise — enough to flip a MoE top-8 boundary on some seeds (cosine 0.96 → fail). Nibbles are instead splatted into bf16 lanes as 128+n with one LOP3 (0x4300 | n), HSUB2 against 128+z (exact), HMUL2 by the bf16 scale — bit-identical to the reference's rounding, runs at 1595 GB/s vs 1625 for a pure stream, and makes the router logits bit-match cublas.
  2. Barrier state persists across launches — the first launch worked and the second deadlocked until the block-local sense was seeded from the global cell.
  3. Global fp32 atomics are ~150 ops/µs-slow — the MLA weighted-sum pass partitions heads across block groups (188 = 47 l-chunks × 4 head-groups) so the L2 absorbs row re-reads and atomics drop 4×.

The per-stage clock64 stamps (readable via the scratch tensors) were essential since profilers can't see inside a single kernel; scratch/ keeps the dev harnesses (stages.py for the breakdown, dbg8.py for a 20-config sweep, stream_bench.py for the dequant-variant DRAM rates).
