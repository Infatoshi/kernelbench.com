The FP8 GEMM is implemented in solution.py and passes check.py.

Approach

Triton tl.dot on fp8_e4m3 — uses native FP8 tensor-core MMA (fp32 accumulate), then applies per-channel weight_scale and writes bf16.

Stride-aligned padding — the main performance issue for K=4127 was row stride 4127 (% 128 = 31), which caused ~6× slower memory access vs aligned stride 4096. When K or strides aren’t 128-aligned, inputs are padded to K_pad = ceil(K/128)*128 with zeros in the tail columns (no effect on the dot product). Padded weight and activation buffers are cached on the model to avoid per-forward allocation.
