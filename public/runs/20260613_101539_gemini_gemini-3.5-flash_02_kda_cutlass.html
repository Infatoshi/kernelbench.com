Architectural Design & Optimization Strategy

1.

assistant

Intra-Chunk Parallelization via Block-Wise Inversion In the naive KDA recurrence, computing the unit

assistant

assistant

a loop is highly inefficient. I developed a highly parallel block-wise unit-triangular matrix inversion algorithm that

assistant

doubles the step size at each iteration ($1, 2, 4, 8, 16, 32$). This

assistant

reduces the sequential complexity from $63$ loops to just $6$ fully batched matrix multiplications that saturate

assistant

  1. **Vectorized Interaction Matmuls (The $\exp(g)
assistant

\cdot \exp(-g)$ Trick)** Instead of materializing large 3D tensors for $\exp(g

assistant

_i - g_j)$ or doing expensive element-wise loops, I mathematically simplified the interactions: $$A = \

assistant

beta \odot \left( (k e^g) \cdot (k e^{-g})^T \right)$$

assistant

This maps the entire intra-chunk decay directly to standard, optimized 2D matrix multiplications ($64 \times 128 \times

assistant

64$), which PyTorch Inductor can compile with near-peak compute density.

  1. **Sequential Inter-Chunk Recurrence as
assistant

a Custom Triton Kernel** The sequential recurrence across chunks is strictly sequential and represents the primary latency bottleneck of linear attention. Standard

assistant

torch.compile is unable to eliminate the loop overheads and intermediate memory allocations of the recurrence state $S$. I

implemented the sequential loop over $NT$ chunks inside a custom register-resident Triton kernel (recurrence_kernel) executed with `num

_stages=1`. By keeping the state $S$ ($128 \times 128$ float32) entirely in registers

/SRAM, we achieved:
