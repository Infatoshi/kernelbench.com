  2. Single Triton megakernel launch per step() — full [KDA×3 + MLA] + MoE + norms + residuals.
  3. Fused W4A16 dequant-GEMV — int4 unpack + group-128 asymmetric dequant in the GEMV (no full bf16 materialize).
  4. MLA absorb — scores/context via latent cache, not full kv_b materialization over L.
  5. Atomic grid barriers (arrive + release flag) for multi-SM phases; private top-k per program.

Still slower than baseline. Correctness and megakernel structure are in place; the gap is mostly:

  • Software spin barriers (~45 phases × 4 blocks)
  • Scalar-ish GEMV / KDA state loops vs cuBLAS
  • O(L) MLA loops at long context

Main knobs for more speed

  1. CUDA cooperative launch (grid.sync) instead of atomic spin barriers
  2. Tighter GEMV (shared x, better coalescing, more warps)
  3. Vectorized MLA score/context over L
  4. Fewer barriers by fusing more independent work

The timed path is one real fused kernel (no CUDA graphs / torch.compile / multi-kernel decode loop). Next big wins are barrier + GEMV quality, not more Python fusion.
